1. Problem (6)¶

\begin{equation}\label{eq:problem1} \begin{aligned} \mnmz_{\bm p}\ &\bm c^\top \bm p \\ \st\ &\sum_{i=1}^np_i=1,\\ &0\le p_i,\quad\forall i=1,\dots,n,\\ &\norm{\bm p-\bm p^0}_{k}\le\eps. \end{aligned} \end{equation}

1.1. Comparison with a solver¶

Since the objective function of the problem above does not depend on the order of the components of $\bm p$, without loss of generality we can assume that the vector $\bm c$ is sorted. As a result of the fact that $\bm p$ has to be a probability distribution, an increase in some components of $\bm p$ has to lead to a decrease of some other components of $\bm p$ by the same margin.To minimize the objective function, one has to decrease components of $\bm p$ corresponding to the components of $\bm c$ with high values, i.e. components with large index since the vector $\bm c$ is sorted. To demonstrate this behavior, we set $\bm p^{0}$ as uniform distribution and vector $\bm c$ is generated from a normal distribution and then sorted.

n  = 20;
p0 = ones(n)/n;
c  = sort(rand(Normal(0,1), n));
  = 0.05;

p_l1_solver = Projections.minimize_linear_on_simplex_exact(p0, c, , 1);

p_l1   = Projections.minimize_linear_on_simplex_l1(p0, c, );
p_l2   = Projections.minimize_linear_on_simplex_l2(p0, c, );
p_lInf = Projections.minimize_linear_on_simplex_lInf(p0, c, );

To check if the p_l1 solution given by our algorithm is correct, we compare it with the p_l1_solver solution given by the ECOS solver. To keep the presentation simple, we compare results only fo the $l_{1}$ norm. Below you can see the norm of the differences between these two solutions

p_l1_solver - p_l1  = 7.829923411e-11

Moreover, below we compare values of objective functions for both solutions

c'*p_l1_solver = 0.01613292478
c'*p_l1        = 0.01613292479

and we also check if the solutions are feasible

Solver solution: 
   p = 1:       1.0 = 1
   p - p0  : 0.05000000001  0.05
Our solution: 
   p = 1:       1.0 = 1
   p - p0  : 0.05  0.05

It is clear that our solution is almost identical to the solution given by the solver. The following figures show the comparison of the original uniform distribution $\bm p^{0}$ with distributions given by the projection for $l_{1}$, $l_{2}$ and $l_{\infty}$ norm.

The main results are presented in the figure below, where we compare the time complexity of the algorithms for the increasing length of the vector $\bm p^{0}$. The following settings of inputs and parameters were used for the experiments

  • $\bm p^{0} \sim \mathit{Uniform}(0,1)$,

  • $n \in [10^{2},10^{5}]$,
  • $\varepsilon = 0.1$,

where $n$ is the length of the vector $\bm p^{0}$. For each $n$ we ran each algorithm ten times and averaged execution times. In the figure above we see (almost) linear dependence of time on the data size $n$. For the $l_{2}$ norm we provide the comparison of our method and the algorithm from [Philpott 2018], see figure below. Note that the computational complexity for [Philpott 2018] is quadratic.

2. Problem (8)¶

\begin{equation}\label{eq:problem4} \begin{aligned} \mnmz_{\bm p,\bm q, r}\ &\frac12\norm{\bm p - \bm p^0}^2 + \frac12\norm{\bm q - \bm q^0}^2 + \frac12(r-r^0)^2\\ \st\ &\sum_{i=1}^n p_i=\sum_{j=1}^m q_j,\\ &0\le p_i\le C_1,\quad\forall i=1,\dots,n,\\ &0\le q_j\le C_2r,\quad\forall j=1,\dots,m. \end{aligned} \end{equation}

2.2. Time comparison¶

As for the problem (6), in this section we provide a simple comparison of the time complexity of the presented algorithms. The following settings of inputs and parameters were used for the experiments

  • $\bm p^{0} \sim \mathit{Normal}(0,1)$,
  • $\bm q^{0} \sim \mathit{Normal}(0,1)$,
  • $\bm r^{0} \sim \mathit{Uniform}(0,1)$,
  • $N \in [10^{2},10^{5}]$,
    • $n = \lceil 0.3\cdot N \rceil$
    • $m = \lceil 0.7\cdot N \rceil$
  • $C1 = 0.7$
  • $C2 = 0.9$

where $n$ is the length of the vector $\bm p^{0}$ and $m$ is the length of the vector $\bm q^{0}$. For each $N$ we ran each algorithm ten times and averaged execution times. We use two different methods of finding the root of the function: bisection and secant methods (both methods are implemented in Roots package)

In the figure above we see (almost) linear dependence of time on the data size $N$. We can also see, that the algorithm using the secant method is faster than the one using the bisection method. On the other hand, the algorithm using the bisection method has a guaranteed convergence.
